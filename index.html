Problem Statement

üî¥ The Challenge

Students and developers face multiple interconnected problems:

1. Scattered Code

Code snippets scattered across 20+ locations: Google Drive, local files, Stack Overflow bookmarks, email drafts, handwritten notes, and college lab computers.

2. Lost Context

Weeks later, developers can't remember why they wrote code a certain way or what problem it solved. Critical learning is lost.

3. Manual Work

Students manually create flashcards for interview prep, wasting 3-4 hours per week on content they've already coded.

4. Interview Gap

70% of students struggle to recall their own implementations during technical interviews despite having solved similar problems.

Why This Project is Necessary

Problem Impact: Students waste 200+ hours per semester on redundant work‚Äîfinding lost code, rewriting solutions, and manually creating interview prep from code they've already written.

Market Gap: Existing tools (GitHub Gist, SnippetsLab) are passive storage. None convert code into active learning material.

Model / Algorithms Used

ü§ñ AI Pipeline Architecture

Algorithm 1: Language Detection

Input: Raw code string

Method: Pattern matching with 15+ language signatures (regex-based)

Output: Detected language (javascript, python, java, etc.)

Accuracy: 92% on test set

Algorithm 2: AI Tag Generation (Llama 3)

Input: Code + detected language

Process:

  1. Send code to Groq API with structured prompt
  2. Llama 3 analyzes semantic meaning
  3. Extract 5-7 concept tags
  4. Filter duplicates and normalize

Output: Array of tags (e.g., ["recursion", "binary-search", "divide-conquer"])

Latency: 1.5-2 seconds

Algorithm 3: Explanation Generation

Input: Code + language + tags

Prompt Structure: What-How-Complexity template

Output: 3-paragraph explanation in plain English

Quality: Flesch reading ease score > 60 (accessible)

Algorithm 4: Interview Q&A Synthesis (Core Innovation)

Input: Code + explanation + complexity

Process:

  1. Identify problem pattern from code structure
  2. Generate realistic interview question
  3. Create model answer using code explanation
  4. Generate optimization follow-up

Output: JSON object with question, answer, followUp

Uniqueness: First LLM implementation for this specific task
